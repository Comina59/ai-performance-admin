# AI+绩效考核系统交互需求文档（研发执行版）

- 日期：2026-05-27
- 版本：v1.1
- 目标：让研发按页面与按钮即可完成开发，不讨论技术实现与数据模型。

## 1. 系统目标

围绕一条固定业务链路：

员工发起当月绩效 -> 领导审批评分项 -> 员工自评 -> 领导独立评分 -> 差异判定 -> 触发面谈 -> 输出改进计划 -> 周期结束。

## 2. 角色

1. 员工：发起、填写自评、查看结果、参与面谈、确认改进计划。
2. 管理者：审批、独立评分、面谈、确认改进计划。
3. HR（可选首版）：仅查看进度和统计，不参与打分。

## 3. 全局状态（前后端统一）

1. 草稿中
2. 待领导审批
3. 待员工自评
4. 待领导评分
5. 差异判定中
6. 待绩效面谈（触发时）
7. 待改进计划确认
8. 已完成

## 4. 员工端需求（按页面与按钮）

## 4.1 页面：我的绩效（列表页）

展示字段：周期（月）、当前状态、截止日期、待办动作。

按钮：
1. `新建本月绩效`
2. `继续填写`
3. `查看结果`

规则：
1. 同一员工同一月份只允许一个绩效单。
2. 如果已有草稿，点击“新建本月绩效”提示“本月已存在草稿，是否继续编辑”。

## 4.2 页面：评分项配置（员工发起）

页面元素：
1. 评分项表格：评分项名称、权重。
2. 顶部实时显示：权重合计（xx/100）。

按钮：
1. `新增评分项`
2. `删除评分项`
3. `保存草稿`
4. `提交领导审批`

点击行为：
1. 点“新增评分项”：表格新增一行空项。
2. 点“删除评分项”：删除当前行并重算总权重。
3. 点“保存草稿”：保存当前内容，状态保持“草稿中”。
4. 点“提交领导审批”：先校验，通过后状态改为“待领导审批”。

校验与提示：
1. 评分项名称不能为空。
2. 权重必须大于0。
3. 权重合计必须等于100，否则禁止提交并提示“权重合计需为100分，当前为xx分”。

## 4.3 页面：自评填写

前置条件：仅在“待员工自评”状态可编辑。

页面元素：
1. 每个评分项一行：权重、员工自评分、成果描述、附件上传。
2. 顶部显示：自评总分（自动汇总）。

按钮：
1. `AI润色描述`
2. `AI检查内容完整性`
3. `保存草稿`
4. `提交自评`

点击行为：
1. 点“AI润色描述”：只改文字表达，不改分数。
2. 点“AI检查内容完整性”：返回缺失项提示（如“缺少可量化成果”）。
3. 点“提交自评”：校验通过后提交，状态变“待领导评分”。

校验与提示：
1. 每项自评分不得超过该项权重。
2. 自评分不得小于0。
3. 成果描述不能为空。
4. 不通过时标红对应行并提示原因。

## 4.4 页面：结果与差异

前置条件：领导评分完成后可查看。

展示内容：
1. 员工分、领导分、差值（分项+总分）。
2. 是否触发面谈及触发原因。

按钮：
1. `查看面谈安排`（触发时显示）
2. `确认查看`

## 4.5 页面：改进计划确认

前置条件：面谈结束后。

展示内容：
1. 改进项
2. 负责人
3. 截止日期
4. 验收标准

按钮：
1. `确认计划`
2. `申请调整`（填写原因后提交给管理者）

## 5. 管理者端需求（按页面与按钮）

## 5.1 页面：团队绩效工作台

列表分区：
1. 待我审批
2. 待我评分
3. 待我面谈

按钮：
1. `去审批`
2. `去评分`
3. `发起面谈`
4. `查看详情`

## 5.2 页面：评分项审批

展示内容：
1. 员工填写的评分项与权重。
2. 权重合计。

按钮：
1. `审批通过`
2. `驳回并填写原因`

点击行为：
1. 点“审批通过”：状态改为“待员工自评”。
2. 点“驳回并填写原因”：弹窗必填驳回原因，提交后状态回“草稿中”。

## 5.3 页面：领导评分

前置条件：员工已提交自评。

展示内容：
1. 左侧：员工自评内容与证据。
2. 右侧：领导独立评分输入区（每项独立填写分数和评语）。

按钮：
1. `AI生成评语草案`
2. `AI检查评语风险`
3. `保存草稿`
4. `提交领导评分`

关键要求：
1. 领导评分输入区默认不带入员工分数（只展示员工内容供参考）。
2. 每项分数不得超过该项权重。

提交后行为：
1. 系统自动执行差异判定，状态进入“差异判定中”。
2. 判定完成后：
   - 若触发阈值，状态改“待绩效面谈”。
   - 若未触发，直接进入“待改进计划确认”或“已完成”（按是否要求强制计划）。

## 5.4 页面：差异判定结果

系统判定规则固定：
1. 任一分项差异比例 >= 30%（差值/该项权重）
2. 或总分差 >= 20分
满足任一即触发。

页面展示：
1. 触发条款（命中了哪条）
2. 差异最大Top项
3. 推荐面谈要点

按钮：
1. `一键发起面谈`
2. `稍后处理`

## 5.5 页面：绩效面谈

展示内容：
1. AI面谈提纲（可编辑）
2. 面谈纪要输入框
3. 面谈总结

按钮：
1. `AI生成面谈提纲`
2. `AI生成面谈总结`
3. `生成改进计划`
4. `完成面谈`

点击行为：
1. 点“生成改进计划”：生成可编辑计划清单。
2. 点“完成面谈”：状态改“待改进计划确认”。

## 6. AI功能边界（避免产品风险）

1. AI只做建议，不自动改分。
2. AI不允许修改员工/领导已确认分值。
3. 所有AI生成内容都可编辑后再提交。
4. 提交时要标记“本段内容由AI生成并人工确认”。

## 7. 消息通知（首版必须）

1. 员工提交审批后，通知管理者“有待审批绩效项”。
2. 审批通过后，通知员工“请完成自评”。
3. 员工提交自评后，通知管理者“请完成评分”。
4. 触发面谈后，通知员工与管理者“已触发绩效面谈”。
5. 生成改进计划后，通知员工“请确认改进计划”。

## 8. 首版验收用例（研发联调标准）

1. 员工权重合计不为100时，无法提交审批。
2. 员工任一项自评分超过权重时，无法提交自评。
3. 领导可在不继承员工分的情况下完成独立评分。
4. 命中“单项30%或总分20分”时，系统自动进入待面谈。
5. 面谈完成后，员工能看到并确认改进计划。
6. 全流程每一步状态变化在列表页可见。

## 9. MVP不做项（防止范围失控）

1. 不做薪酬自动联动。
2. 不做跨周期复杂对比分析。
3. 不做多人联合评分（仅员工+直属管理者）。
4. 不做移动端独立App（先Web/H5）。
